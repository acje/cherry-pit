# adr-fmt

ADR template and link-integrity validator for cherry-pit. Single source of
truth for all invariant ADR governance rules. Validates all ADRs under
`docs/adr/` against the rule catalog and regenerates README index files.

## Invocation

```bash
cargo run -p adr-fmt
```

Auto-discovers `docs/adr/` by walking up from the current directory looking for `docs/adr/GOVERNANCE.md`.

Use `--report` to print a computed children index (reverse-link navigation without stored backlinks) in addition to the normal lint and README generation:

```bash
cargo run -p adr-fmt -- --report
```

Use `--guidelines` to print the complete ADR guidelines document generated from the rule catalog and configuration:

```bash
cargo run -p adr-fmt -- --guidelines
```

`--report` and `--guidelines` are mutually exclusive.

## Side Effects

Running `adr-fmt` is **not read-only**. It regenerates README files on every invocation:

- Domain READMEs (e.g., `docs/adr/cherry/README.md`)
- Stale archive README (`docs/adr/stale/README.md`)
- Top-level hub (`docs/adr/README.md`)

Only files whose first line is `<!-- Generated by adr-fmt. Do not edit. -->` are overwritten. Hand-edited files are protected and skipped with a warning.

## Exit Codes

- `0` — Lint complete. Warnings may be present. **Parse stderr for diagnostics** — exit 0 does not mean "no issues."
- `1` — Infrastructure error (missing config, no domain directories found).

## SSOT Architecture

- **`adr-fmt` (code)** — invariant rules: template, naming, vocabulary, lifecycle, links
- **`adr-fmt.toml`** — configurable: domains, crate mappings, rule params, stale directory
- **`--guidelines` output** — generated complete reference (replaces prescriptive governance sections)
- **`GOVERNANCE.md`** — rationale, process, judgment only

## Rule Catalog

All rules emit warnings. `adr-fmt` is advisory.

### Template (T001-T015)

| Rule | Description |
|------|-------------|
| T001 | H1 title present |
| T002 | Date field present |
| T003 | Last-reviewed field present (required for all tiers) |
| T004 | Tier field present |
| T005 | Status section with status line present |
| T006 | Status value recognized; no parenthetical annotations |
| T007 | Related section has at least one relationship (no orphans, no placeholders) |
| T008 | Context section present |
| T009 | Decision section present |
| T010 | Consequences section present |
| T011 | Code block exceeds 20 lines |
| T012 | Amendment date >= creation date; valid ISO 8601 |
| T013 | *(reserved — Retirement section covers all terminal states)* |
| T014 | Section ordering: Status, Related, Context, Decision, Consequences (+ Retirement for stale) |
| T015 | Prose section below minimum word count (configurable via `min_words` param) |

### Link (L001-L009)

| Rule | Description |
|------|-------------|
| L001 | Dangling link — target ADR not found in any domain |
| L003 | Supersedes verb without matching Superseded-by status on target |
| L004 | Cross-domain reference to unmigrated ADR |
| L006 | Legacy relationship verb used (Depends on, Extends, Illustrates, etc.) |
| L007 | Stale reference — target ADR is in stale archive |
| L008 | Root verb target does not match the ADR's own ID |
| L009 | Root and References verbs coexist — Root ADRs may not also use References |

### Naming (N001-N004)

| Rule | Description |
|------|-------------|
| N001 | Filename matches `PREFIX-NNNN-kebab-slug.md` pattern |
| N002 | Number in filename matches H1 title ID |
| N003 | Slug is valid lowercase kebab-case |
| N004 | Domain prefix not found in configuration |

### Structure (S004-S006)

| Rule | Description |
|------|-------------|
| S004 | Stale ADR missing required Retirement section (at least 10 words) |
| S005 | Active ADR has Retirement section (forbidden outside stale/) |
| S006 | Terminal-status ADR (Rejected/Deprecated/Superseded) not in stale directory |

### Index (I001-I003) — Internal

Internal self-checks verifying generated README consistency. Not shown in user-facing output.

| Rule | Description |
|------|-------------|
| I001 | ADR file exists on disk but is missing from domain README index |
| I002 | Domain README references an ADR that does not exist on disk |
| I003 | Domain README hyperlink target does not resolve to a file |

## Configuration

`docs/adr/adr-fmt.toml` — domain definitions, stale directory config, and rule parameters.

Key configurable values:

- `[[domains]]` — prefix, name, directory, description, crates, foundation
- `[stale]` — stale archive directory name
- `[[rules]]` — rule catalog with optional `[rules.params]` (e.g., T015 `min_words = 10`) and `internal = true`

## ADR Template

Run `cargo run -p adr-fmt -- --guidelines` for the complete template specification and all required fields.

## Workflow

After creating or editing any ADR:

1. Run `cargo run -p adr-fmt`
2. Parse stderr output for warnings
3. Fix all reported issues
4. Re-run to confirm clean output
5. Commit the ADR **and** any regenerated README files together
